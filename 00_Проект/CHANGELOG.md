# CHANGELOG

## Release 1.0 — FOUNDATION (2026-07-08)

### Добавлено
- Полная файловая структура Проекта (28 разделов верхнего уровня).
- Комплект управляющих документов КП-001…КП-007, версия 1.0, статус «Утверждено».
- Реестры: документов, версий, терминов (`00_Проект/02_Реестр`).
- Базовый набор шаблонов ШБ-001…ШБ-008 (`17_Шаблоны`).
- СТД-001 «Диагностика теплообменных аппаратов», версия 1.0.
- Заготовки разделов `18_ИИ_Агенты` для шести ИИ-агентов (ChatGPT, DeepSeek, Claude, Gemini, Qwen, Grok).
- Отчёт Sprint_001 с итогами формирования учредительной структуры.

### Известные ограничения
- Разделы 02–16 (кроме 01_Стандарты_СТД, где есть СТД-001) созданы как структура и ожидают содержательного наполнения.
- Промпты (`07_Промпты`) ещё не наполнены рабочими сценариями.
- СТД-002 и СТД-003 находятся в статусе «Черновик».

### Обновлено (09.07.2026)
- ПРМ-001 (ChatGPT, СТД), ПРМ-003 (ChatGPT, РГЛ), ПРМ-005 (ChatGPT, ИНС) — версия 1.1: добавлен «Режим Б» — автономный подбор оборудования/узла/типовой операции и значений по открытым данным (ГОСТ/СП, типовые каталоги, отраслевая практика), когда исходных данных по объекту нет. Во всех случаях сохранено обязательное требование КП-003: количественные параметры и требования безопасности не могут получить статус «Утверждено» без проверки человеком-экспертом.

## Release 1.1 (в подготовке) — наполнение 07_Промпты

### Добавлено
- 13 рабочих промптов для ИИ-агентов, сгруппированных по типу целевого документа (СТД, РГЛ, ИНС, ЭНЦ, FAQ, RAG, сквозная проверка качества): ChatGPT (черновики документов, FAQ), Claude (структурирование СТД, сборка RAG-карточек), DeepSeek (поиск противоречий, аудит, проверка RAG), Qwen (нормализация терминологии), Gemini (анализ фото/видео оборудования), Grok (сортировка входящих данных).
- Промпты зарегистрированы в `Реестр_документов.xlsx` (коды ПРМ-001…ПРМ-013).
- Индекс раздела `07_Промпты/README.md` с таблицей соответствия «раздел → тип документа → промпты».
