# Промпт: ChatGPT — черновик стандарта диагностики (СТД)

| Поле | Значение |
|---|---|
| Агент | ChatGPT |
| Целевой тип документа | СТД (01_Стандарты_СТД) |
| Версия промпта | 1.1 |
| Статус | Утверждено |
| Связанный шаблон | `17_Шаблоны/ШБ-001_СТД.docx` |
| Связанный регламент | КП-003 (Регламент работы ИИ) |

## Когда использовать
Два режима работы с этим промптом:
- **Режим А (есть исходные данные):** нужно быстро получить черновик СТД по оборудованию, для которого уже есть паспорт, регламент завода-изготовителя или опыт эксплуатации.
- **Режим Б (автономный подбор):** исходных данных по конкретному объекту нет — ChatGPT сам выбирает распространённый тип теплотехнического оборудования и параметры диагностики на основе открытых источников (общедоступные инженерные справочники, ГОСТ/СНиП, типовые характеристики производителей), чтобы получить рабочий черновик, который затем проверяется экспертом и привязывается к реальному оборудованию объекта.

## Промпт — Режим А (копировать целиком, заполнив поля в квадратных скобках)

```
Ты — технический писатель инженерной документации по эксплуатации теплотехнического оборудования.
Составь черновик стандарта диагностики строго по структуре шаблона ниже.
Не придумывай числовые нормы, допуски и параметры, если они не указаны в исходных данных —
в этих местах пиши "[ТРЕБУЕТ ПРОВЕРКИ ЭКСПЕРТОМ]".

СТРУКТУРА (обязательна к соблюдению один в один):
1. Область применения
2. Контролируемые параметры (таблица: Параметр | Норма | Метод измерения | Периодичность)
3. Признаки отклонений
4. Порядок диагностики (нумерованный список шагов)
5. Ссылки (на 10_Каталог_неисправностей, 11_Физика_процессов, 08_Карточки_оборудования)

ИСХОДНЫЕ ДАННЫЕ:
Тип оборудования: [указать]
Место эксплуатации / объект: [указать]
Известные параметры и нормы: [вставить, что есть]
Известные признаки неисправностей: [вставить, что есть]

ТРЕБОВАНИЯ К СТИЛЮ:
- Формально-деловой стиль, без воды.
- Термины — только из 09_Технический_словарь; новый термин — вынеси отдельным списком в конце
  с пометкой "требует добавления в словарь".
- Не используй фразы вида "как правило", "обычно" — только конкретные значения или явная пометка
  об отсутствии данных.
```

## Промпт — Режим Б: автономный подбор оборудования и параметров по открытым данным

```
Ты — технический писатель инженерной документации по эксплуатации теплотехнического оборудования.
Данных по конкретному объекту нет. Самостоятельно выбери один распространённый тип
теплотехнического оборудования (например: кожухотрубный или пластинчатый теплообменник,
водогрейный котёл, циркуляционный насос, узел учёта тепловой энергии — выбери один) и составь
по нему черновик стандарта диагностики строго по структуре шаблона ниже, используя ТОЛЬКО
общедоступные открытые данные: действующие ГОСТ/СП/СНиП, типовые технические характеристики
из открытых каталогов производителей, общепринятые в отрасли справочные значения.

СТРУКТУРА (обязательна к соблюдению один в один):
1. Область применения
2. Контролируемые параметры (таблица: Параметр | Норма | Метод измерения | Периодичность)
3. Признаки отклонений
4. Порядок диагностики (нумерованный список шагов)
5. Ссылки (на 10_Каталог_неисправностей, 11_Физика_процессов, 08_Карточки_оборудования)

ОБЯЗАТЕЛЬНЫЕ ПРАВИЛА ДЛЯ ЭТОГО РЕЖИМА:
- Для каждого числового значения в разделе «Контролируемые параметры» укажи в отдельной
  колонке или сноской конкретный тип открытого источника, на основании которого оно приведено
  (например: "типовой диапазон по открытым каталогам производителей", "ГОСТ Р ХХХХ-ХХ",
  "общепринятое отраслевое значение"). Если источник — общее отраслевое знание без конкретного
  документа, так и пиши, не выдумывай номер ГОСТа, если не уверен в его существовании.
- В начале документа, сразу после метаданных, добавь блок:
  "ВНИМАНИЕ: документ составлен по открытым данным без привязки к конкретному оборудованию
  объекта. Все значения — типовые, ТРЕБУЮТ ПРОВЕРКИ И ПРИВЯЗКИ К ФАКТИЧЕСКОМУ ПАСПОРТУ
  ОБОРУДОВАНИЯ ОБЪЕКТА ПЕРЕД УТВЕРЖДЕНИЕМ (см. КП-002)."
- Раздел "Признаки отклонений" и "Порядок диагностики" тоже строй на общепринятой инженерной
  практике диагностики для выбранного типа оборудования, не на данных конкретного объекта.

ТРЕБОВАНИЯ К СТИЛЮ:
- Формально-деловой стиль, без воды.
- Термины — только из 09_Технический_словарь; новый термин — вынеси отдельным списком в конце
  с пометкой "требует добавления в словарь".
- Не используй фразы вида "как правило", "обычно" без указания источника этого обобщения.
```

## После получения ответа
1. Скопировать результат в новый файл по шаблону `ШБ-001_СТД.docx`, статус — «Черновик» (`19_Черновики`).
2. Заполнить метаданные (код, версия 0.1, ответственный, дата). Для Режима Б в поле «Источник» указать "ИИ (ChatGPT), открытые данные — требует привязки к объекту".
3. Передать на рецензию согласно КП-002. Поля «[ТРЕБУЕТ ПРОВЕРКИ ЭКСПЕРТОМ]» и блок предупреждения из Режима Б — обязательны к закрытию до перехода в статус «На редакции»: эксперт должен либо подтвердить типовые значения для конкретного оборудования объекта, либо заменить их фактическими.
4. Новые термины — зарегистрировать в `Реестр_терминов.xlsx`.
5. Документ, составленный в Режиме Б, не может получить статус «Утверждено», пока эксперт не подтвердит соответствие фактическому оборудованию объекта (КП-003, раздел «Ограничения»: количественные параметры обязательно проверяются человеком перед утверждением).
