# Промпт: Claude — структурирование и сборка СТД в docx

| Поле | Значение |
|---|---|
| Агент | Claude |
| Целевой тип документа | СТД (01_Стандарты_СТД) |
| Версия промпта | 1.0 |
| Статус | Утверждено |
| Связанный шаблон | `17_Шаблоны/ШБ-001_СТД.docx` |
| Связанный регламент | КП-003 |

## Когда использовать
Когда черновик СТД уже прошёл рецензию (статус «На редакции» → «Утверждено») и нужно привести его к финальному оформлению — проверить структуру, метаданные и подготовить файл .docx для публикации в `01_Стандарты_СТД`.

## Промпт

```
Ты работаешь с документом стандарта диагностики (СТД) для корпоративной базы знаний
Knowledge Factory «Теплоконтур». Прими на вход текст черновика и выполни:

1. Проверь соответствие структуре: Область применения → Контролируемые параметры →
   Признаки отклонений → Порядок диагностики → Ссылки. Если раздел отсутствует — добавь
   пустой раздел с пометкой "требует заполнения", не удаляй и не переставляй местами.
2. Проверь, что каждая строка таблицы «Контролируемые параметры» содержит все 4 поля
   (Параметр, Норма, Метод измерения, Периодичность). Не заполненные поля помечай явно,
   не оставляй пустыми молча.
3. Проверь, что метаданные документа заполнены: Код, Версия, Статус, Ответственный, Дата,
   Источник (в данном случае — "ИИ (Claude) + [имя рецензента]").
4. Собери итоговый файл .docx по образцу структуры действующего СТД-001, сохрани версию
   документа как переданную во входных данных (не меняй номер версии самостоятельно).
5. В конце добавь список изменений относительно предыдущей версии, если она была передана
   на вход.

Не добавляй, не удаляй и не изменяй технические значения (нормы, допуски, параметры) —
твоя задача форматирование и структурная проверка, а не редактирование фактов.

ВХОДНЫЕ ДАННЫЕ:
[вставить текст черновика СТД]
[вставить предыдущую версию, если применимо, для формирования списка изменений]
```

## После получения ответа
1. Сверить итоговый .docx с чек-листом КП-002, раздел «Критерии готовности к утверждению».
2. Зарегистрировать новую версию в `Реестр_версий.xlsx` с указанием "Источник: ИИ (Claude)".
3. Переместить документ в целевой раздел (`01_Стандарты_СТД/<код>`), включить в ближайший релиз согласно КП-005.
