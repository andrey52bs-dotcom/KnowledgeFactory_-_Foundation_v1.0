# Промпт: ChatGPT — черновик регламента обслуживания (РГЛ)

| Поле | Значение |
|---|---|
| Агент | ChatGPT |
| Целевой тип документа | РГЛ (02_Регламенты_РГЛ) |
| Версия промпта | 1.1 |
| Статус | Утверждено |
| Связанный шаблон | `17_Шаблоны/ШБ-002_РГЛ.docx` |
| Связанный регламент | КП-003 |

## Когда использовать
Два режима работы с этим промптом:
- **Режим А (есть исходные данные):** нужен черновик РГЛ по конкретному узлу/системе, для которых уже есть эксплуатационные данные, руководство завода-изготовителя или опыт персонала.
- **Режим Б (автономный подбор):** данных по конкретному узлу объекта нет — ChatGPT сам выбирает распространённый тип узла теплотехнического оборудования и формирует типовой регламент обслуживания на основе открытых источников (ГОСТ/СП, типовая эксплуатационная документация производителей, общепринятая отраслевая практика). Черновик Режима Б обязательно проходит проверку экспертом и специалистом по охране труда перед утверждением.

## Промпт — Режим А

```
Ты — технический писатель, готовящий регламент планового обслуживания оборудования.
Составь черновик строго по структуре:

1. Метаданные (Код РГЛ-XXX, Версия 0.1, Статус Черновик, Ответственный)
2. Периодичность обслуживания (таблица: Операция | Периодичность | Исполнитель)
3. Порядок выполнения работ (нумерованные шаги: подготовка → основные операции → контроль результата)
4. Требования безопасности
5. Используемые материалы и инструмент

ИСХОДНЫЕ ДАННЫЕ:
Оборудование / узел: [указать]
Известная периодичность и объём работ: [вставить]
Требования по охране труда, если есть: [вставить]

ОГРАНИЧЕНИЯ:
- Периодичность и объём работ указывай только на основании переданных данных.
  При отсутствии данных пиши "[ТРЕБУЕТ ПОДТВЕРЖДЕНИЯ ОТВЕТСТВЕННЫМ ЗА ОБОРУДОВАНИЕ]",
  не предлагай "типовые" значения от себя.
- Раздел "Требования безопасности" не сокращай и не обобщай — если данных недостаточно,
  явно укажи, что раздел требует заполнения профильным специалистом по ОТ.
```

## Промпт — Режим Б: автономный подбор узла и регламента по открытым данным

```
Ты — технический писатель, готовящий регламент планового обслуживания оборудования.
Данных по конкретному узлу объекта нет. Самостоятельно выбери один распространённый узел
теплотехнического оборудования (например: циркуляционный насос, запорная арматура,
теплообменный аппарат, узел учёта тепловой энергии — выбери один) и составь по нему
типовой черновик регламента обслуживания строго по структуре, используя ТОЛЬКО общедоступные
открытые источники: действующие ГОСТ/СП, типовую эксплуатационную документацию производителей,
общепринятую в отрасли практику планового обслуживания.

1. Метаданные (Код РГЛ-XXX, Версия 0.1, Статус Черновик)
2. Периодичность обслуживания (таблица: Операция | Периодичность | Исполнитель)
3. Порядок выполнения работ (нумерованные шаги: подготовка → основные операции → контроль результата)
4. Требования безопасности
5. Используемые материалы и инструмент

ОБЯЗАТЕЛЬНЫЕ ПРАВИЛА ДЛЯ ЭТОГО РЕЖИМА:
- Для периодичности каждой операции указывай тип открытого источника (например: "типовая
  периодичность по эксплуатационной документации производителей", "ГОСТ Р ХХХХ-ХХ",
  "общепринятая отраслевая практика"). Не придумывай номер ГОСТа, если не уверен в его
  существовании — в этом случае пиши "общепринятая отраслевая практика" без ссылки на номер.
- В начале документа, сразу после метаданных, добавь блок:
  "ВНИМАНИЕ: документ составлен по открытым данным без привязки к конкретному узлу объекта.
  Все значения периодичности и объёма работ — типовые, ТРЕБУЮТ ПРОВЕРКИ И ПРИВЯЗКИ К
  ФАКТИЧЕСКОМУ ОБОРУДОВАНИЮ ОБЪЕКТА ПЕРЕД УТВЕРЖДЕНИЕМ (см. КП-002)."
- Раздел "Требования безопасности" формируй на основе общих отраслевых норм охраны труда для
  выбранного типа работ, но с явной пометкой "ТРЕБУЕТ ПРОВЕРКИ СПЕЦИАЛИСТОМ ПО ОХРАНЕ ТРУДА
  ПЕРЕД ИСПОЛЬЗОВАНИЕМ" — этот раздел ни при каких условиях не публикуется без такой проверки.
```

## После получения ответа
1. Разместить черновик в `19_Черновики` по шаблону ШБ-002. Для Режима Б в поле «Источник» указать "ИИ (ChatGPT), открытые данные — требует привязки к объекту".
2. Обязательно направить раздел "Требования безопасности" на отдельную проверку специалисту по охране труда, независимо от общей рецензии по КП-002 — это требование действует для обоих режимов и не может быть пропущено.
3. Для Режима Б — эксперт по оборудованию обязан подтвердить или заменить типовые значения периодичности на фактические перед переходом в статус «На редакции».
4. Зарегистрировать документ в `Реестр_документов.xlsx`.
