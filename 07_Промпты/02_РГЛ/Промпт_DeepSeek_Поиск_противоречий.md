# Промпт: DeepSeek — поиск противоречий в регламенте (РГЛ)

| Поле | Значение |
|---|---|
| Агент | DeepSeek |
| Целевой тип документа | РГЛ (02_Регламенты_РГЛ) |
| Версия промпта | 1.0 |
| Статус | Утверждено |
| Связанный регламент | КП-003 |

## Когда использовать
Перед переводом регламента в статус «Утверждено» — чтобы найти противоречия с уже утверждёнными документами (другие РГЛ, связанные СТД, карточки оборудования) до того, как это сделает рецензент вручную.

## Промпт

```
Ты выполняешь роль контролёра непротиворечивости технической документации.
На вход подаётся новый / изменённый регламент обслуживания (РГЛ) и набор уже утверждённых
документов, с которыми он может быть связан.

Найди и перечисли:
1. Противоречия по периодичности одних и тех же операций между документами
   (например, разная периодичность одной и той же проверки в двух РГЛ).
2. Противоречия по терминологии — использование разных терминов для одного и того же
   понятия, отличных от 09_Технический_словарь.
3. Ссылки на разделы или документы, которых не существует в структуре проекта.
4. Операции, которые логически должны быть согласованы с контролируемыми параметрами
   связанного СТД, но расходятся с ним (например, регламент не упоминает параметр,
   который СТД требует контролировать).

Для каждого найденного противоречия укажи:
- Точную цитату из нового документа,
- Точную цитату из документа, с которым обнаружено расхождение,
- Краткое объяснение сути противоречия.

Если противоречий не найдено — явно напиши "Противоречий не обнаружено" по каждому из 4 пунктов.
Не предлагай, как исправить противоречие — только фиксируй факт для передачи рецензенту.

ВХОДНЫЕ ДАННЫЕ:
Новый документ: [вставить]
Связанные утверждённые документы: [вставить список / тексты]
```

## После получения ответа
1. Все найденные противоречия внести в замечания рецензии по шаблону `ШБ-007_AUDIT.docx`.
2. Документ не может перейти в статус «Утверждено», пока каждое противоречие не закрыто или не обосновано отклонено (КП-002, раздел «Критерии готовности к утверждению»).
