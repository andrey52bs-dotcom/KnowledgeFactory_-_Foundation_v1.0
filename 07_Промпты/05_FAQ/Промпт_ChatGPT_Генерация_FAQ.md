# Промпт: ChatGPT — генерация FAQ на основе СТД/РГЛ/ИНС

| Поле | Значение |
|---|---|
| Агент | ChatGPT |
| Целевой тип документа | FAQ (05_FAQ) |
| Версия промпта | 1.0 |
| Статус | Утверждено |
| Связанный шаблон | `17_Шаблоны/ШБ-005_FAQ.docx` |
| Связанный регламент | КП-003 |

## Когда использовать
После утверждения нового СТД, РГЛ или ИНС — чтобы автоматически предложить набор вопросов, которые реально будет задавать персонал, и краткие ответы на основе утверждённого документа.

## Промпт

```
Ты помогаешь превратить формальный технический документ в раздел FAQ для персонала эксплуатации.
На вход подаётся утверждённый документ (СТД / РГЛ / ИНС).

Составь 5–10 вопросов, которые реально задаст инженер или оператор, читая этот документ
на практике (не общие вопросы вида "что такое X", а практические: "что делать, если...",
"как часто нужно...", "можно ли пропустить..."). Для каждого вопроса дай краткий ответ
(2-3 предложения), основанный ИСКЛЮЧИТЕЛЬНО на содержании входного документа.

Формат вывода — для каждого вопроса отдельно:
Вопрос: [текст]
Краткий ответ: [2-3 предложения]
Источник: [раздел исходного документа, из которого взят ответ]

Если для типичного практического вопроса в документе нет ответа — не придумывай его,
а укажи вопрос отдельным списком "Вопросы без ответа в исходном документе — требуют
уточнения у эксперта".

ВХОДНОЙ ДОКУМЕНТ: [вставить текст утверждённого СТД/РГЛ/ИНС]
```

## После получения ответа
1. Каждую пару «вопрос-ответ» оформить отдельным файлом или записью по шаблону ШБ-005.
2. Обязательно проверить, что "Источник" в ответе указывает на реальный раздел документа — рецензент сверяет соответствие перед утверждением.
3. Список "Вопросы без ответа" передать ответственному за раздел — это сигнал для доработки исходного СТД/РГЛ/ИНС.
