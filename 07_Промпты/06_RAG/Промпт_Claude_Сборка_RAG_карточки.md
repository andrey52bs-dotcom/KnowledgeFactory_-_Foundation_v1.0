# Промпт: Claude — сборка RAG-карточки из утверждённого документа

| Поле | Значение |
|---|---|
| Агент | Claude |
| Целевой тип документа | RAG-карточка (06_RAG) |
| Версия промпта | 1.0 |
| Статус | Утверждено |
| Связанный шаблон | `17_Шаблоны/ШБ-008_RAG.json` |
| Связанный регламент | КП-003 |

## Когда использовать
После утверждения любого документа (СТД, РГЛ, ИНС, ЭНЦ, FAQ, кейс) — чтобы нарезать его на одну или несколько RAG-карточек, пригодных для поиска и прямой вставки в ответ ИИ-агента конечному пользователю.

## Промпт

```
Ты готовишь RAG-карточки (Retrieval-Augmented Generation) на основе утверждённого документа
базы знаний Knowledge Factory «Теплоконтур». Карточка должна быть самодостаточной: пригодной
для использования вне контекста исходного документа.

Правила нарезки:
1. Один самостоятельный факт, правило или параметр — одна карточка. Не объединяй в одну
   карточку несколько несвязанных фактов.
2. Поле "summary" — 1-3 предложения, которые можно вставить в ответ пользователю без изменений
   и без потери точности.
3. Поле "parameters" заполняй только если в исходном документе есть конкретное числовое значение
   с единицей измерения — не переноси общие описания в это поле.
4. Поле "related_faults" / "related_solutions" заполняй только если в исходном документе есть
   явная ссылка на конкретный раздел 10_Каталог_неисправностей / 12_Типовые_решения.
5. Строго заполни JSON-схему шаблона ШБ-008_RAG.json целиком, включая provenance
   (author, ai_agent_used: "Claude", prompt_reference: код этого промпта, date).

Не добавляй в карточку ничего, чего нет в исходном документе. Если факт кажется важным,
но неполным (например, упомянут параметр без нормы) — не дополняй его собственными
предположениями, оставь поле пустым.

ИСХОДНЫЙ ДОКУМЕНТ (утверждённый, с указанием кода и версии): [вставить текст]
```

## После получения ответа
1. Каждую полученную карточку сохранить отдельным .json файлом в `06_RAG` с именем `RAG-XXXX_<краткое_название>.json`.
2. Присвоить карточке следующий свободный ID и зарегистрировать в `Реестр_документов.xlsx`.
3. Карточка получает статус «Утверждено» только после того, как рецензент сверил "summary" с исходным документом дословно по смыслу (КП-002).
