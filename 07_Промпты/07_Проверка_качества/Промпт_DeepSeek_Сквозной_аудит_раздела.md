# Промпт: DeepSeek — сквозной аудит противоречий по разделу

| Поле | Значение |
|---|---|
| Агент | DeepSeek |
| Область применения | Любой раздел с несколькими документами (01–16) |
| Версия промпта | 1.0 |
| Статус | Утверждено |
| Связанный шаблон | `17_Шаблоны/ШБ-007_AUDIT.docx` |
| Связанный регламент | КП-003, КП-007 (раздел 4 — периодический аудит) |

## Когда использовать
При плановом квартальном аудите раздела (обязанность ответственного за раздел, КП-007) — чтобы получить первичный список потенциальных проблем перед ручной проверкой.

## Промпт

```
Ты выполняешь плановый аудит непротиворечивости и актуальности набора документов одного
раздела базы знаний.

На вход подаётся список документов раздела (тексты или краткие выжимки) с их метаданными
(код, версия, статус, дата последнего изменения).

Проверь и перечисли:
1. Документы с истёкшим сроком актуальности (дата последнего изменения старше 1 года
   от текущей даты аудита) — как кандидаты на пересмотр.
2. Пары документов, которые ссылаются друг на друга, но версия одного из них была изменена
   позже другого — риск устаревшей перекрёстной ссылки.
3. Документы, дублирующие содержание друг друга (совпадающие по смыслу разделы) —
   нарушение принципа "единственный источник истины" (КП-001, раздел 4).
4. Документы без заполненных обязательных метаданных (код, версия, статус, ответственный).

Результат оформи как таблицу с колонками: № | Документ | Замечание | Критичность
(Критично / Средне / Незначительно) — формат совместим с шаблоном ШБ-007_AUDIT.

ВХОДНЫЕ ДАННЫЕ:
Дата аудита: [указать]
Раздел: [указать]
Список документов с метаданными: [вставить]
```

## После получения ответа
1. Результат вставляется в раздел «3. Выявленные замечания» шаблона `ШБ-007_AUDIT.docx`.
2. Замечания с критичностью «Критично» — инициируют пересмотр документа согласно КП-002 в первую очередь.
3. Итоговый аудит подписывается человеком-аудитором (ИИ формирует только черновик замечаний, КП-003).
