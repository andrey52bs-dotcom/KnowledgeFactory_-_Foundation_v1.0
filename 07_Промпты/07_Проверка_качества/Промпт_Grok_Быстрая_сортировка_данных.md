# Промпт: Grok — быстрая сортировка входящих данных по разделам проекта

| Поле | Значение |
|---|---|
| Агент | Grok |
| Область применения | Первичная обработка новых материалов (любой раздел) |
| Версия промпта | 1.0 |
| Статус | Утверждено |
| Связанный документ | КП-006 (Структура проекта) |

## Когда использовать
Когда поступил большой неструктурированный набор материалов (заметки, переписка, фото, отчёты с объекта) и нужно быстро понять, в какие разделы структуры проекта это должно лечь, прежде чем передавать на детальную обработку другим агентам или людям.

## Промпт

```
Ты выполняешь первичную сортировку входящих материалов по разделам корпоративной базы знаний
Knowledge Factory «Теплоконтур». Список разделов и их назначение:

01_Стандарты_СТД — параметры и методы диагностики
02_Регламенты_РГЛ — периодичность и порядок обслуживания
03_Инструкции_ИНС — пошаговые действия персонала
04_Энциклопедия — теория, физика процессов
05_FAQ — вопросы-ответы
08_Карточки_оборудования — паспортные данные конкретных единиц оборудования
10_Каталог_неисправностей — описания неисправностей и признаков
12_Типовые_решения — готовые решения проблем
13_Кейсы_объектов — истории конкретных случаев на объектах
14_Фотоархив / 15_Видеоархив — медиаматериалы
16_Нормативные_документы — внешние нормативные документы

На вход подаётся набор разрозненных материалов. Для каждого фрагмента материала укажи:
- Краткое содержание (1 предложение)
- Наиболее подходящий раздел (один из списка выше)
- Уверенность (Высокая / Средняя / Низкая)
- Если материал относится сразу к нескольким разделам — укажи все и поясни, что в какой раздел
  должно попасть

Не пытайся сразу оформить материал по шаблону нужного раздела — только сортировка и маршрутизация.

ВХОДНЫЕ МАТЕРИАЛЫ: [вставить весь необработанный массив]
```

## После получения ответа
1. Результат сортировки передаётся ответственным за соответствующие разделы.
2. Материалы с «Низкой» уверенностью — на ручной разбор, не отправлять напрямую в обработку другим ИИ-агентом.
3. Сама сортировка не публикуется как документ — это внутренний рабочий шаг перед КП-002.
