# 07_Промпты

Промпты для ИИ-агентов, используемые при разработке контента согласно **КП-003 (Регламент работы ИИ)**.
Сгруппированы по типу целевого документа (требование КП-003, раздел 5 «Хранение промптов»).

## Структура раздела

| Папка | Тип документа | Промпты |
|---|---|---|
| `01_СТД` | Стандарты диагностики | ChatGPT (черновик), Claude (структурирование и сборка) |
| `02_РГЛ` | Регламенты обслуживания | ChatGPT (черновик), DeepSeek (поиск противоречий) |
| `03_ИНС` | Инструкции | ChatGPT (черновик) |
| `04_ЭНЦ` | Энциклопедия | ChatGPT (черновик), Qwen (нормализация терминологии) |
| `05_FAQ` | FAQ | ChatGPT (генерация вопросов-ответов) |
| `06_RAG` | RAG-карточки | Claude (сборка карточки), DeepSeek (проверка точности) |
| `07_Проверка_качества` | Сквозные, не привязаны к одному типу | DeepSeek (аудит раздела), Gemini (анализ фото/видео), Grok (сортировка входящих данных) |

## Как использовать промпт

1. Открыть нужный файл — в шапке указаны агент, целевой тип документа, версия промпта и связанный шаблон/регламент.
2. Скопировать текст из блока «## Промпт», заполнить поля в квадратных скобках своими данными.
3. Выполнить в соответствующем ИИ-агенте (см. `../18_ИИ_Агенты`).
4. Выполнить шаги из раздела «После получения ответа» — они обязательны, это не рекомендация, а часть процесса КП-002/КП-003.

## Правила работы с этим разделом (из КП-003)

- Каждый промпт версионируется отдельно (см. таблицу метаданных в каждом файле) — при существенном изменении логики промпта версия увеличивается по правилам КП-004.
- Любой документ, созданный с использованием промпта из этого раздела, обязан в своих метаданных содержать ссылку на конкретный файл промпта (поле «Источник: ИИ (агент) + промпт»).
- Ни один из промптов в этом разделе не позволяет ИИ-агенту самостоятельно перевести документ в статус «Утверждено» — это ограничение соблюдается сознательно на уровне формулировок промптов.

## Дальнейшее наполнение

Раздел покрывает основные сценарии по каждому типу документа на старте (Release 1.1). По мере появления новых практических сценариев — промпты добавляются сюда же, в соответствующую подпапку, с регистрацией в `../00_Проект/02_Реестр/Реестр_документов.xlsx`.
