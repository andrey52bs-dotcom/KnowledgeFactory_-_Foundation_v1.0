# 13_Кейсы_объектов

Практические кейсы с конкретных объектов.

## Наполнение
2 иллюстративных примера, демонстрирующих полный цикл «признак → диагностика по СТД → неисправность по каталогу → типовое решение» — по шаблону ШБ-006:

| Код | Кейс | Цепочка документов |
|---|---|---|
| КЕЙС-001 | Рост перепада давления на теплообменнике | СТД-001 → НСП-001 → РЕШ-001 |
| КЕЙС-002 | Кавитация циркуляционного насоса | ИНС-001 → СТД-003 → НСП-005 → РЕШ-005 |
| КЕЙС-003 | Рост температуры уходящих газов котла (газ/жидкое топливо) | СТД-002 → НСП-002 → РЕШ-002 |
| КЕЙС-004 | Накипь на ТЭН электрокотла | СТД-004 → НСП-008 → РЕШ-008 |
| КЕЙС-005 | Зашлаковывание колосниковой решётки | СТД-005 → НСП-010 → РЕШ-010 |

Теперь для всех пяти типов оборудования есть иллюстративный пример полного цикла.

**Важно:** оба кейса — иллюстративные примеры, не привязаны к реальному объекту (поле «Объект» — плейсхолдер). Реальный кейс должен содержать фактические данные: название объекта, даты, показания приборов, исполнителя.

Кейс КЕЙС-002 изначально рассматривался как кандидат материала для раздела «Типичные ошибки» ИНС-001, но в итоговую v1.0 этого документа вошли не он, а реальные документированные отраслевые случаи из открытых источников — более строгое основание для статуса «Утверждено» (см. КП-002). КЕЙС-002 остаётся как самостоятельный иллюстративный пример.
